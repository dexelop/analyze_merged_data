# 입력 데이터 스펙

## 파일 위치

```
jsons/{company}/
├── *_분개장.json
├── *_손익계산서.json
├── *_회사등록_기본사항.json
├── *_자동전표-신카-국세청.json
└── *_매입매출전표입력.json
```

---

## 1. 분개장.json

**핵심 컬럼:**

| 컬럼 | 설명 | 예시 |
|------|------|------|
| `da_date` | 전표일자 | "20240101" |
| `cd_acctit` | 계정코드 | "81100" |
| `nm_acctit` | 계정명 | "복리후생비(판)" |
| `nm_gubun_bungae` | 차/대변 | "차변", "대변" |
| `mn_bungae1` | 차변금액 | 39000 |
| `mn_bungae2` | 대변금액 | 0 |
| `no_exter2` | 증빙유형코드 | 88 (신용카드) |
| `nm_trade` | 거래처명 | "롯데카드주식회사" |
| `cd_trade` | 거래처코드 | "099644" |
| `nm_remark` | 적요 | "예수금잔액정리" |
| `ty_gubn` | 차대구분코드 | 3(차변), 4(대변) |

**증빙유형 (no_exter2):**
- `0`: 수기입력
- `86`: 과세 세금계산서
- `88`: 신용카드
- `89`: 현금영수증
- `90`: 자동전표-통장

## 4. 자동전표-신카-국세청.json (신용카드 자동전표)

**핵심 컬럼:**

| 컬럼 | 설명 | 예시 |
|------|------|------|
| `da_sbook` | 사용일자 | "20240101" |
| `nm_trade` | 가맹점명 | "주식회사 에스알" |
| `mn_total` | 총금액 | 36500 |
| `mn_vat` | 부가세 | 0 |
| `ty_jungstat` | 전표상태 | 2 (전표확정) |
| `ty_gongjea` | 공제구분 | 2 |
| `cd_acctit_cha` | 차변계정코드 | "81200" |
| `cd_acctit_dae` | 대변계정코드 | "10100" |
| `nm_acctit_cha` | 차변계정명 | "(판)여비교통비" |
| `nm_acctit_dae` | 대변계정명 | "현금" |

**전표상태 (ty_jungstat):**
- `1`: 확정가능
- `2`: 전표확정
- `3`: 확정제외
- `4`: 중복전표
- `5`: 미추천
- `6`: 삭제전표

## 5. 매입매출전표입력.json

**핵심 컬럼:**

| 컬럼 | 설명 | 예시 |
|------|------|------|
| `da_date` | 전표일자 | "20240115" |
| `ty_mth2` | 전표유형코드 | 51 (과세매입) |
| `nm_trade` | 거래처명 | "..." |
| `mn_sum` | 공급가액 | 1000000 |
| `mn_tax` | 부가세 | 100000 |

**전표유형 (ty_mth2):**
- 매출: 11~25
- 매입: 51~62

## 파일 선택 우선순위

같은 종류의 파일이 여러 개일 때:
1. **가장 최신 타임스탬프** 파일 사용
2. 파일명 형식: `{timestamp}_{module}_smarta_{type}_{name}.json`
